# Bulba Office

2D-игра на TypeScript + Phaser 3 + Vite. Офис с Бульбазаврами-сотрудниками: ходишь по карте, разговариваешь с коллегами и смотришь их «демо» на проекторе. Карта и зоны проходимости размечаются в редакторе [Tiled](https://www.mapeditor.org/).

## Запуск

```bash
npm install
npm run dev      # дев-сервер с автообновлением
npm run build    # проверка типов + прод-сборка в dist/
npm run preview  # локальный просмотр прод-сборки
```

## Геймплей

- Выбираешь персонажа на старте — им управляешь, с остальными разговариваешь.
- Стрелки / WASD — ходить. Спрайт зеркалится в сторону движения.
- Пробел рядом с NPC — диалог. Реплика печатается побуквенно в облачке над NPC.
- Меню вопросов — компактный ряд «таблеток» внизу. Один из вопросов — «Есть апдейты с прошлого демо?»: NPC отвечает и предлагает показать слайды.
- «Давай» — слайд появляется на экране проектора в переговорке (всегда пиксельный). Кнопка на экране разворачивает слайды на весь экран: листание, пиксельный/обычный вид.

## Структура

```
public/assets/
  location.png          фон локации
  location-overlay.png  копия фона для «верха двери» (рисуется поверх игрока)
  <role>.png            спрайты персонажей (dev, dev-fe, analyst, lead, qa,
                        product-owner, designer)
  sample_1..3.png       запасные слайды (если у персонажа нет своих)
  <id>_1.png ...        слайды конкретного персонажа (nikita_1.png, ...)
  office.tmj            карта Tiled (фон + слой коллизий)
src/
  main.ts                  конфиг Phaser
  scenes/BootScene.ts      загрузка ассетов
  scenes/WorldScene.ts     карта, игрок, NPC, движение, взаимодействие,
                           overlay двери, связка диалога/облачка/проектора
  data/characters.ts       персонажи: роли, позиции, поворот, реплики, число слайдов
  entities/sprites.ts      ключи спрайтов, кэш картинок, отрисовка портретов
  ui/CharacterSelect.ts    экран выбора персонажа
  ui/Dialogue.ts           меню вопросов и переходы (через колбэки)
  ui/SpeechBubble.ts       облачко с побуквенной печатью над NPC
  ui/Projector.ts          экран проектора в игре: слайд + кнопка «во весь экран»
  ui/SlideViewer.ts        полноэкранный просмотр слайдов (DOM)
  ui/slides.ts             пути слайдов персонажа + запасные слайды
```

## Слайды («демо»)

- Файлы кладутся в `public/assets/` с именем `<id>_<номер>.png`, где `<id>` — поле персонажа из `characters.ts` (`nikita`, `artur`, ...), номер с 1.
- Сколько слайдов у персонажа — поле `slideCount` в `characters.ts`.
- Если ни одного файла нет, показываются `sample_1..3.png`.

## Как править карту проходимости (Tiled)

1. Открой `public/assets/office.tmj` в Tiled.
2. Слой `background` (Image Layer) — фон-референс `location.png`, по нему рисуешь.
3. Слой `collision` (Object Layer) — прямоугольники/полигоны поверх стен и непроходимых зон.
   - Где нарисован объект — стена. Где пусто — проход (например, дверные проёмы).
   - В проёме двери коллизии быть не должно, иначе игрок упрётся.
4. Сохрани (`Ctrl+S`) — Vite подхватит изменения, перезагрузи страницу.

Слой `doors` (Object Layer) — двери локации, по одной точке-объекту на дверь. **Имя точки =
`id` соседней локации**, к которой ведёт дверь (`main-office`, `chill-zone`, `data-center`,
`parking`). Каждая точка работает в обе стороны:
- появление: войдя из соседа `X`, игрок встаёт в точку с именем `X`;
- зона выхода: стоя рядом с точкой `X` (квадрат `EXIT_ZONE_HALF` вокруг неё), видишь кнопку
  перехода в `X`.

Слой `spawns` (Object Layer) — стартовые позиции персонажей, по одной точке на персонажа.
**Имя точки = `id` персонажа** из `characters.ts` (`nikita`, `artur`, …). Где стоит NPC —
там и его точка; выбранный игроком персонаж стартует в своей точке. Координат в `characters.ts`
нет — только в этом слое. NPC попадает в ту локацию, чей `locationIndex` указан (по умолчанию 0).

Слой `interactions` (Object Layer) — интерактивные объекты. Сейчас точка `tv` в чилл-зоне:
подойдя к ней, игрок видит «Пробел — выбрать игру», открывается меню выбора игры, а из него —
полноэкранное окно игры. Доступны крестики-нолики (`TicTacToe.ts`) и Bulba Jump (`BulbaJump.ts`,
аналог Doodle Jump в антураже склада: посылки-платформы, конвейеры, батуты, прыгает спрайт
выбранного персонажа). Точку `tv` двигаешь в Tiled под телевизор. Новая игра = пункт в массиве
`GAMES` в `GameMenu.ts` + её окно-компонент, открываемый в `WorldScene.openGame`.

В `locations.ts` остаётся только топология (`exits: [{ to }]`) — вся геометрия дверей в Tiled.
У парковки карты нет: её переходы — пункты меню (фаст-тревел). Если в локации нет двери к
соседу, из которого пришёл фаст-тревел, игрок встаёт в первую дверь карты.

## Где что менять

- Реплики, имена, роли, позиции, поворот (`faceRight`), число слайдов — `src/data/characters.ts`.
- Скорость, дистанция взаимодействия, глубины слоёв (`DEPTH`) — константы вверху `src/scenes/WorldScene.ts`.
- Скорость печати и размеры облачка — константы вверху `src/ui/SpeechBubble.ts`.
- Прямоугольник экрана проектора и размер пикселя — `SCREEN` / `BLOCK` вверху `src/ui/Projector.ts`.
- «Верх двери»: координаты задаются содержимым `location-overlay.png` (прозрачно везде, кроме перемычки двери); слой добавляется в `WorldScene.buildDoorOverlay`.
